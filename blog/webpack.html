# Webpack 使用

# 入门

webpack 需求

  • 开发过程中，代码保存自动刷新页面就好了 -热更新
  • 写css样式，或者js代码后自动处理兼容问题 -代码编译
  • 项目上线，一键压缩代码，图片处理 -压缩打包服务

代码转换、文件优化、代码分割、模块合并、自动刷新、代码校验、自动发布。

webpack本质上是一个javascript应用程序的静态模块打包器。

什么是模块

  • ES2015的import语句
  • CommonJs require语句
  • AMD define和require语句
  • Css/Scss/Less文件中的import
  • 样式或html文件的图片、字体文件 安装

npm install webpack webpack-cli --save-dev

  • nrm ls 查看有哪些源可以使用（nrm需要安装）
  • nrm use taobao 切换源
  • nrm test taobao 测速
  • 如安装失败，推荐切换成国内镜像

# 打包js

步骤

  • 新建一个项目，然后npm init -y
  • 新建一个js，写一些代码
  • 在命令行直接 npx webpack ...js 打包完成

如果想自己配置一下webpack，就新建一个webpack.config.js

//基本配置
const path=require("path");
module.exports={
	mode:'production',
	//入口
	entry:"./index.js",
	//出口
	output:{
		filename:"index.js",
		path:path.resolve(__dirname,"dist")
	}
}

# babel

js的编译器，es6+向es5转换

npm install @babel/core @babel/preset-env babel-loader -D

但标准引入的全局变量，部分原生对象，例如，新增的原型链上的方法 Promise symbol set。babel是没办法转换的。

第一种解决方式：用@babel/polyfill来解决，但它也有问题，比如开发类库，UI组件时，会全局变量的污染。

第二种解决方式：用@babel/plugin-transform-runtime @babel/runtime-corejs3,以闭包方式注入，保证全局环境不被污染。

# 例子

webpack 4入门 步骤

  • npm init -y
  • npm install webpack --save-dev
  • npm install webpack-cli --save-dev
  • 新建src/index.js,写一些js代码
  • package.json里配置dev prod
  • npm install babel-preset-env --save-dev,npm install @babel/preset-env --save-dev
  • 新建.bablerc,配置presets,配置根据babel版本不一样，最好根据官网来配置
  • npm install babel-loader --save-dev
  • 新建src/a.js 写一下代码，然后在index.js里引入a.js,再配置.bablerc
  • 新建dist/index.html,引入main.js
  • 要用css还是需要新建webpack.config.js，配置
  • npm install --save-dev css-loader style-loader。
	部分代码如下
	//webpack.config.js
	module.exports = {
	  module: {
	    rules: [
	      {
	        test: /\.css$/i,
	        use: ['style-loader', 'css-loader'],
	      },
	    ],
	  },
	};
	//package.json
	"prod":"webpack --mode production --module-bind js=babel-loader",
	"dev":"webpack --mode development --module-bind js=babel-loader"
	//.babelrc
	{
		"presets":[["@babel/preset-env",{"modules":false}]]
	}

# 进阶

treeShaking 摇树优化，在webpack4中只要是production，就会自动优化，而在webpack3以前 需要安装ugligyjsWebpackPlugin,注意源码必须遵循 ES6 的模块规范 (import & export)，如果是 CommonJS 规范 (require) 则无法使用。

# 生产环境和开发环境不同配置

开发环境 生产环境
devServer treeShaking
sourceMap 代码压缩
接口代理 proxy 提供公共代码

一般用三个配置文件，webpack.prod.js,webpack.dev.js,webpack.base.js分别代表生产，开发，公用的代码，然后 借助一个工具 webpack.merge把文件合并。

步骤：

  • 安装 npm install webpack.merge
  • 在配置文件中引入 const merge=require("webpack-merge")
  • 导出时用merge合并一下，module.export=merge(baseConfig,devConfig) , baseConfig和devConfig是引入的配置文件
  • 在package.json中，更改启动文件，例如 "build":"webpack --config ./webpack.dev.js"

# 打包优化

三种方式：

  • 入口配置：entry多入口 webpack.ProviePlugin
  • 抽取公共代码 splitchunksPlugins
  • 动态加载：按需加载，懒加载，@babel/plugin-syntax-dynamic-import

一般都是加载第三方库时，例如jquery。
