# Webpack 使用

# 入门

webpack 需求

  • 开发过程中，代码保存自动刷新页面就好了 -热更新
  • 写css样式，或者js代码后自动处理兼容问题 -代码编译
  • 项目上线，一键压缩代码，图片处理 -压缩打包服务

代码转换、文件优化、代码分割、模块合并、自动刷新、代码校验、自动发布。

webpack本质上是一个javascript应用程序的静态模块打包器。

什么是模块

  • ES2015的import语句
  • CommonJs require语句
  • AMD define和require语句
  • Css/Scss/Less文件中的import
  • 样式或html文件的图片、字体文件 安装

npm install webpack webpack-cli --save-dev

  • nrm ls 查看有哪些源可以使用（nrm需要安装）
  • nrm use taobao 切换源
  • nrm test taobao 测速
  • 如安装失败，推荐切换成国内镜像

# 打包js

步骤

  • 新建一个项目，然后npm init -y
  • 新建一个js，写一些代码
  • 在命令行直接 npx webpack ...js 打包完成

如果想自己配置一下webpack，就新建一个webpack.config.js

//基本配置
const path=require("path");
module.exports={
	mode:'production',
	//入口
	entry:"./index.js",
	//出口
	output:{
		filename:"index.js",
		path:path.resolve(__dirname,"dist")
	}
}

# babel

js的编译器，es6+向es5转换

npm install @babel/core @babel/preset-env babel-loader -D

但标准引入的全局变量，部分原生对象，例如，新增的原型链上的方法 Promise symbol set。babel是没办法转换的。

第一种解决方式：用@babel/polyfill来解决，但它也有问题，比如开发类库，UI组件时，会全局变量的污染。

第二种解决方式：用@babel/plugin-transform-runtime @babel/runtime-corejs3,以闭包方式注入，保证全局环境不被污染。

# 进阶

treeShaking 摇树优化，在webpack4中只要是production，就会自动优化，而在webpack3以前 需要安装ugligyjsWebpackPlugin,注意源码必须遵循 ES6 的模块规范 (import & export)，如果是 CommonJS 规范 (require) 则无法使用。

# 生产环境和开发环境不同配置

开发环境 生产环境
devServer treeShaking
sourceMap 代码压缩
接口代理 proxy 提供公共代码

一般用三个配置文件，webpack.prod.js,webpack.dev.js,webpack.base.js分别代表生产，开发，公用的代码，然后 借助一个工具 webpack.merge把文件合并。

步骤：

  • 安装 npm install webpack.merge
  • 在配置文件中引入 const merge=require("webpack-merge")
  • 导出时用merge合并一下，module.export=merge(baseConfig,devConfig) , baseConfig和devConfig是引入的配置文件
  • 在package.json中，更改启动文件，例如 "build":"webpack --config ./webpack.dev.js"

# 构建SPA

步骤

  • 核心功能
  • 编译依赖
  • 动态更新
  • Tree shaking
  • 提取css
  • 上线

操作js步骤

1.新建一个文件夹spa-webpack，初始化 npm init -y,安装 npm install webpack --save-dev,新建目录src/index.js

2.添加开发环境"dev":"webpack --mode development"，启动 npm run dev (安装cli)

3.在dist 文件夹新建index.html,引入打包的main.js

4.在src里新建components/sync、async、common,第一步是对js的优化，webpack的tree shaking一般有效，但不能在有函数式作用域，类似loadsh有效果的，需要（webpack-deep-scope-analysis-plugin）插件。或者把类似loadsh的库直接解构完，就不需要其他插件。

操作css步骤

1.安装npm install --save-dev css-loader

2.配置webpack.config.js

3.npm i purgecss-webpack-plugin -D

4.npm install --save-dev mini-css-extract-plugin,它和style-loader冲突，使用时要把style-loader的删掉。

部分代码

	console.log("sync");
	document.getElementById("app").innerHTML=`<h1 class="${item.test}">hello</h1>`

const isarray=function(){
	console.log("isarray");
}
export{
	sync,
	isarray
}

//webpack.config.js
const path = require('path');
const glob = require('glob');
const PurgeCSSPlugin = require('purgecss-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');
const PATHS = {
  src: path.join(__dirname, './dist/*.html')
}
module.exports = {
  module: {
      rules: [
        {
          test: /\.css$/i,
          use:[
			// MiniCssExtractPlugin.loader,
			 // {loader:'style-loader'},用MiniCssExtractPlugin时去掉
			  {loader:'css-loader',
			  options:{
				  modules:{
					  localIdentName:"[name]__[local]-[hash:base64:5]"
						}
				  },
			  },
          ],
        },
      ],
	
    },
	plugins: [
		new MiniCssExtractPlugin(),
	    //new PurgeCSSPlugin({
	    //  paths: glob.sync(`${PATHS.src}/**/*`,  { nodir: true }),
	    //}),用css tree shaking的时候打开
	  ]
};

切换开发、上线环境

1.新建config/webpack.development.js,webpack.production.js

2.npm install webpack-merge --save-dev

3.配置webpack.config.js

4.npm install html-webpack-plugin --save-dev

const {merge}=require("webpack-merge");
const argv=require("yargs-parser")(process.argv.slice(2));
const __mode=argv.mode||"development";
const __mergeConfig=require(`./config/webpack.${__mode}.js`)
const __modeflag=(__mode=="production"?true:false);
const HtmlWebpackPlugin = require('html-webpack-plugin')
webpackConfig= {
  module:{},
   plugins: [
   		new MiniCssExtractPlugin({
   			filename:__modeflag?"styles/[name].[hash:5].css":"styles/[name].css",
   			chunkFilename:__modeflag?"styles/[id].[hash:5].css":"styles/[id].css"
   		}),
   		new HtmlWebpackPlugin({  
   		  filename: 'index.html',
   		  template: 'src/index.html'
   		})
   	  ]
   };
 }
 module.exports=merge(__mergeConfig,webpackConfig)
