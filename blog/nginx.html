# Nginx的反向代理与负载均衡

  • 什么是反向代理与负载均衡
    • 反向代理：反向代理（Reverse Proxy）实际运行方式是指以代理服务器来接受连接请求，然后将请求转发给网络上的服务器集群，具体哪台服务器返数据我们不知道。正向代理，比如我们国内访问国外网站，直接访问访问不到，可以通过一个代理服务器，请求发到代理服务器上，由代理去取到数据返回给我们，我们能知道要访问的地址。
  • nginx负载均衡的实现
    • 我们可以建立多个服务器，这些服务器组成一个服务器集群，然后，当用户访问网站时，先访问一个中间服务器，再让这个中间服务器在服务器集群中选择一个压力较小的服务器，将该访问请求引入该选择的服务器。
  • HTTP Upstream模块
    • 是nginx服务器的一个重要模块。实现在轮询和客户端ip之间实现后端的负载均衡。常用指令有ip_hash指令、server指令和upstream指令等。
    • ip_hash指令，如果客户端请求已经访问了服务器A并登录，那么第二次请求的时候，会将该请求通过哈希算法自动定位到该后端服务器中。
    • server指令主要用于指定服务器的名称和参数。

# 部署nodejs上线步骤

  • 1.安装homebrew
  • 2.brew search nginx 、brew install nginx
  • 3.brew info nginx
  • 4.nginx -v
  • 5.启动 sudo brew services start nginx或nginx
  • 如果安装过jenkins的话，执行下面一步
  • sudo launchctl unload /Library/LaunchDaemons/org.jenkins-ci.plist
  • systemctl start jenkins
  • 6.关闭 sudo brew services stop nginx/nginx
  • 7.nginx -s reload、nginx -s stop
  • 8.打开nginx具体安装目录 查看配置文件 cd /usr/local/etc/ 、ls 、cd nginx 、cat nginx.conf
  • /usr/local/var/www 静态资源文件
  • 验证配置文件 nginx -t -c 自己的配置文件
//test_conf 修改配置文件，完成负载均衡
worker_processes 4;
events{
	worker_connections 1024;
}
http{
	upstream firsttest{
		ip_hash;
		server 111.13.100.92;
		server 111.13.100.92;
	}
	server{
		listen 8080;
		location / {
			proxy_pass http://firsttest;
		}
	}
}

# Nginx的缓存策略

  • 打开nginx 查看配置文件 cd /usr/local/etc/nginx 、cat nginx.conf
  • 默认ETag Last-Modified都是开启的,可以关掉ETag
  • sudo nginx -s reload 重启后，ETag就没有了

跟缓存相关的是 ETag、Last-Modified、、expires、Cache-Control。
