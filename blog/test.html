    # 习题总结

    # ES5与ES6变量提升

    ES5:

    • 1.有变量提升，但因为函数作用域，只会提升到函数的最前面。
    • 2.函数提升比变量优先级高，但如果发现变量提升后没有值，就忽略。

    ES6-9:块作用域 {},会提升到块的最前面，不是函数最前面。

    # this指向

    • ES5 谁调用指向谁，如果找不到就指向window。
    • ES6 箭头函数 ()=>{} this绑定父级的顶级作用域

    # 其他

    • ES5 函数里的优先级比原型链高
    • ES6 暂时性死区，在 {} 块级作用域中，函数要先定义再赋值，否则报错
    • 函数的值是按值传递的，但这个值对应的指向是按引用传递的。
    • ES6 在 {} 块作用域中的函数，只会函数名提升出去，到函数顶部，函数体还在块作用域中

    function addMethod(object,name,fn){
	var old=object[name];//上一次的函数值
	object[name]=function(){
		if(fn.length==arguments.length){
			return fn.apply(this,arguments)
		}else if(typeof old=="function"){
			return old.apply(this,arguments)
		}
	}
}
